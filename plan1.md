1. OverviewA WordPress plugin to automate post creation from an external link database. It uses a "Hidden Template Post" strategy, allowing users to design their post layout using the native Gutenberg Editor.2. Technical ArchitectureTemplate Storage: Custom Post Type lb_template (Internal use only).UI: Admin dashboard to view pending links and a "Template Editor" page.Engine: String-replacement parser that processes Gutenberg block HTML.3. Step-by-Step ImplementationPhase 1: The Template CPT & Hidden LogicRegister CPT lb_template.public => false, show_in_rest => true, supports => ['title', 'editor', 'custom-fields'].Singleton Constraint: Logic to ensure only one "Active Template" exists. If a user tries to create a second, redirect them to the first.The "Starter Pattern":On plugin activation, programmatically create the first lb_template post.Default Content:HTML<div class="wp-block-group">    <h2>LINK_TITLE</h2><p>LINK_DESCRIPTION</p><div class="wp-block-buttons"><div class="wp-block-button"><a class="wp-block-button__link" href="LINK_URL">Read Original Article</a></div></div>    </div>Phase 2: The UI DashboardAdmin Page: "Link Manager".Functionality: * Fetch rows from the custom database table (links).Display a list with a "Generate Post" button for each row.Display a "Link to Template Editor" button at the top.Phase 3: The Generation Engine (The Parser)Implement a class LinkBlog_Generator with a method create_post($link_id):Retrieve Data: Get link data (Title, URL, Desc) from the database via $wpdb.Retrieve Template: Get post_content from the lb_template CPT.Variable Replacement:Search: ['LINK_TITLE', 'LINK_URL', 'LINK_DESCRIPTION']Replace: [$data->title, esc_url($data->url), wp_kses_post($data->desc)]Insertion: Use wp_insert_post() to create a standard post type using the replaced string.Phase 4: Styling & UXTag Auto-Complete (Optional/Recommended): Add a simple JS script to the Gutenberg editor that provides a sidebar with "Click to Copy" tags for the user.Post Meta: Save the original LINK_URL into a post meta field (_original_link_url) for future use or tracking.4. Security & ValidationAll user-inputted template content must pass through force_balance_tags() to prevent broken layouts.The parser must escape the URL replacements using esc_url() to prevent XSS.Use admin_url( 'post.php?post=' . $template_id . '&action=edit' ) to provide a seamless link to the template builder.
